# PaperSpine + Nature 水利课设增强规则

这组文件是我在本地课程设计工作中沉淀出来的一套 Codex skill 增强规则。它不是从零重写 PaperSpine，也不是另起一个孤立的“课程报告 skill”，而是在 PaperSpine 和 Nature-skill 的基础上加了一层更适合水利、生态水工、本科课程设计的调度和表达规则。

最初的问题很具体：写水利报告时，模型容易把所有任务都当成整篇论文工程来跑，哪怕只是改一句话，也会启动完整流程；另一方面，降 AI 痕迹常常停留在换词，忽略了水利报告真正看重的东西，比如高程关系、岸脚防冲、反滤排水、坡面稳定、生态措施与行洪安全之间的先后关系。这个增强包就是为了解决这两个问题。

## 做了什么

核心思路是“主指挥 + 小分工”。

PaperSpine 仍然作为主入口，但它先判断任务大小，再决定用哪条通道：

- 一句话、小段落、局部 Word 修改，走轻量通道；
- 一节文字润色，交给 Nature polishing 或 humanize；
- 表格、公式、模板、Word/PDF 修复，走专项修复；
- 整篇报告重写、结构重建、论文式写作，才启动完整 PaperSpine 流程。

这样做以后，小修不会被迫跑完整 intake、research、citation 和 LaTeX 流程；真正的大项目仍然保留 PaperSpine 的完整骨架。

## 文件结构

```text
skills/
  paper-spine/
    SKILL.md
  nature-polishing/
    static/core/hydraulic-engineering.md
  paper-spine-humanize/
    references/hydraulic-engineering-expression.md
```

三个文件各有分工。

`paper-spine/SKILL.md` 是主指挥。它负责判断任务属于轻量修改、章节润色、报告修复还是完整工作流，并决定是否需要调用 PaperSpine 全流程。这里还加入了模型分派建议：轻量任务可交给 `gpt-5.4`，机械检查可交给 `gpt-5.4-mini`，整篇结构和最终判断仍留给主模型。

`nature-polishing/static/core/hydraulic-engineering.md` 是水利方向的硬约束。它提醒模型不要只把文字写顺，还要看水位、高程、岸脚、反滤层、护坡结构、植物分区、表格计算和工程边界。对生态护岸类报告，它特别强调生态措施要服从行洪断面和边坡稳定。

`paper-spine-humanize/references/hydraulic-engineering-expression.md` 负责中文表达和降 AI 痕迹。它不鼓励机械套用“基于上述……统筹……遵循……原则”这类句式，而是要求把话写成更像学生报告里的工程判断：先说水流、冲刷或渗流压力，再说防冲、排水、护坡的设计顺序，最后说明生态布置的安全边界。

## 为什么要这样分

我不希望一个 skill 变成什么都管的“大杂烩”。PaperSpine 适合做总控，Nature-skill 适合做学术逻辑和水利专业边界，humanize 层适合处理中文语感。把它们分开后，任务小的时候能快，任务大的时候也不会丢掉结构。

这个分法还有一个好处：后续维护比较容易。水利专业规则可以继续放在 hydraulic core 里，降 AI 表达可以继续放在人性化规则里，PaperSpine 主入口只管分派，不需要把所有细节都堆进去。

## 典型例子

原来的句子可能会写成：

> 该断面的重点放在岸脚防冲、反滤排水、坡面稳定和生态布置四个环节，生态措施需服从行洪安全和岸坡稳定要求。

这句话信息没错，但读起来像在列清单。增强后的规则会鼓励改成更自然的工程判断：

> 结合上述水文动力条件，MJ6断面设计先处理岸脚防冲、反滤排水和坡面稳定；生态布置在不影响行洪安全和边坡整体稳定的基础上展开。

差别不在于字更高级，而在于逻辑顺了：先有水文动力条件，再有防冲排水护坡的设计顺序，最后才是生态措施的边界。

## 使用建议

如果只是局部修改，不要启动完整 PaperSpine。

如果用户说“降低 AI 痕迹”，先看是不是句式太模板化，再看有没有工程判断缺失。只换连接词、只改长短句，通常不够。

如果涉及水利报告的数字、表格、高程、坡脚、反滤层、植物分区，不能只润色文字。相关数值要一起闭合，至少要确认旧值没有残留。

如果用户明确说这是大项目，或者要从材料生成整篇报告，再进入 PaperSpine 完整流程。

## 基础来源

本增强包基于两个已有能力继续扩展：

- PaperSpine：负责论文和报告的主流程、结构设计、证据组织和最终交付；
- Nature-skill：负责学术表达、段落逻辑、专业边界和水利方向润色。

这里新增的内容主要面向水利课程设计和生态水工报告。它更像一层本地经验补丁：保留原有框架，同时让它更适合本科课设、Word 成品、中文水利表达和低 AI 痕迹写作。
